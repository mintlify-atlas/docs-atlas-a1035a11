## Key components

| Component | Description |
|-----------|-------------|
| **Access Points** | Entry interfaces for users, systems, and external applications through web-based interfaces, mobile applications, APIs, and direct integrations. |
| **Application Gateway** | Central routing and management layer for request handling, load balancing, and traffic distribution. Ensures secure and efficient communication between external access points and internal services. |
| **Services** | Core business logic and AI processing layer, including the Workflow Engine, Expert AI Reasoning, multiple AI model integrations, and the Human-in-the-Loop framework. |
| **Integrations** | Enterprise connectivity layer with pre-built connectors, API adapters, and data transformation services. Supports over 250 enterprise integrations including SharePoint, Salesforce, ServiceNow, enterprise databases, and email systems. |
| **Security & Compliance** | Enterprise-grade protection through role-based access control, data encryption, audit logging, and regulatory compliance frameworks. |
| **Observability & Monitoring** | Comprehensive visibility into system performance through metrics, logs, and traces across all components. |

## How it works

A workflow starts when a trigger fires — via API call, schedule, event from a third-party app, or manual action. The Workflow Engine executes a directed graph of nodes: each node receives input from the workflow context, processes it, and writes output back. Downstream nodes reference earlier outputs using `{{context.steps.NodeName.output}}` syntax.

```
Trigger (API / Schedule / Event / Manual)
           │
           ▼
   Workflow Engine
   ┌─────────────────────────────────────────┐
   │  Start → AI Node → Human Node → API Node → End │
   │               │          │                     │
   │           Model Hub   Inbox Queue              │
   └─────────────────────────────────────────┘
           │
           ▼
   Output (API Response / File / Notification)
```

## Key solution features

| Concept | Description |
|---------|-------------|
| [Workspaces](/ai-for-process/admin-settings) | Logical groupings of workflows and configurations, organized by teams or projects. Controls what data and workflows are accessible and which permissions apply. |
| [Workflows](/ai-for-process/workflows/overview) | Structured, executable flows of tasks designed to automate a specific business process. Each workflow run executes independently based on provided inputs. |
| [Nodes](/ai-for-process/nodes) | Fundamental building blocks of automation arranged on the visual builder canvas. Categorized by function: AI, Logic, Integration, and Control. |
| [Triggers & Schedules](/ai-for-process/workflows/triggers-and-access) | Enable event-driven automation by initiating workflows automatically via real-time events, scheduled intervals, external API calls, or manual actions. |
| [Context Object](/ai-for-process/workflows/overview) | Central runtime state container that captures inputs, intermediate data, and node-level status for a specific execution. |
| [Prompt Studio](/ai-for-process/prompts) | Dedicated workspace for managing the entire prompt lifecycle, from creation and testing to optimization. Supports multi-model comparison, version control, and 65+ pre-built templates. |
| [Model Hub](/ai-for-process/models) | Centralized interface for full control over the AI model lifecycle. Supports external providers, open-source models, and fine-tuned models. |
| [Inbox (Human-in-the-Loop)](/ai-for-process/inbox) | Centralized queue where human reviewers process tasks requiring manual judgment such as approvals, data input, or content review. |
| [Observability & Analytics](/ai-for-process/settings/monitoring-overview) | Suite including the Workflow Monitor, Audit Logs, and performance dashboards. Tracks success rates, response times, latency, and token consumption. |
| [Security & Administration](/ai-for-process/settings/security-control) | Enterprise-grade controls managing users via RBAC and secure authentication through SSO. |
| [Integrations](/ai-for-process/settings/integrations) | Connectivity layer with pre-built connectors for 120+ third-party services, including Salesforce, Slack, and AWS S3. |
