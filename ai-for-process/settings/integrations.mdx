The **Integrations** feature connects your AI for Process account to third-party services. Once connected, use them from the workflow canvas with the Integration node.

AI for Process supports 120+ integrations with the following authentication methods:

| Auth method | Description |
|-------------|-------------|
| **API** | Authenticate using a token in headers or query parameters. Used for API Key or Access Token integrations. |
| **OAuth2** | Uses the OAuth2 authorization framework to grant limited access without sharing passwords. |
| **Bearer** | Uses a bearer token in the request header, typically within an OAuth2 flow. |
| **Basic Auth** | Authenticates using a username and password in the request header. Supported for ServiceNow, Freshdesk, Snowflake, Amplitude, and Mixpanel. |

<Note>Other authentication methods for certain integrations include Basic, Basic with JWT, OAuth1, and custom authentication defined by the service provider.</Note>

## Access integrations

1. In AI for Process, click **Settings** in the top menu.
2. Click **Integrations** in the left menu.

The **Integrations** page has two tabs:
- **All Integrations**: Lists available integrations grouped by category.
- **Connected**: Lists all configured connections.

Use the **Search** field, **Category** filter, or **Authorization** filter to find specific integrations. Toggle between **List view** and **Tile view** (default) using the display options.

## Add a connection

<Note>You can also add a connection directly from the **Integration node** on the workflow canvas.</Note>

<Steps>
  <Step title="Access Integrations">
    In AI for Process, click **Settings** > **Integrations**.
  </Step>
  <Step title="Select the integration">
    In the **All Integrations** tab, optionally filter by **Category** and **Authorization**, then click the required integration.
  </Step>
  <Step title="Click Add Connection">
    Click **Add Connection** on the integration page.
  </Step>
  <Step title="Configure authorization details">
    Enter a **Connection Name** and configure the **Authorization Details** based on the selected auth type:

    <AccordionGroup>
      <Accordion title="OAuth2">
        Select the configured **Auth Profile** under **Custom** to authenticate the integration. After selecting an auth profile, its credentials populate the corresponding fields. Click **Authorize** to test the integration.
      </Accordion>
      <Accordion title="Bearer Token">
        Enter the bearer token and any related credentials such as Base URL, API Key, or Bot Token. Click **Test** to validate the connection.
      </Accordion>
      <Accordion title="API">
        Enter the **API Key** or **Access Token**. Additional fields may be required based on the provider. Click **Test** to validate the connection.
      </Accordion>
      <Accordion title="Basic Auth">
        Enter the required credentials (for example, Amplitude requires an API key and API secret). Click **Test** to validate the connection.
      </Accordion>
    </AccordionGroup>
  </Step>
  <Step title="Save">
    Click **Save** after testing the connection successfully. A success message confirms the connection.
  </Step>
</Steps>

## Manage connections

In the **Connected** tab, you can view, edit, test, and manage all configured connections.

Each connection entry shows: Connection Name, Added By, Authorization Details, Integration Type, Added on, and an Action to test or enable/disable.

### Edit a connection

1. Access **Integrations** > **Connected** > Select the connection.
2. Click the **Ellipses** icon > **Edit**.
3. Modify the required fields in the **Authorization Details** section.
4. Optionally click **Test** to validate.
5. Click **Save**.

### Delete a connection

1. Access **Integrations** > **Connected** > Select the connection.
2. Click the **Ellipses** icon > **Delete**.
3. Click **Delete** to confirm.

<Warning>Deletion is irreversible and removes all associations of the connection from the platform.</Warning>

### Enable or disable a connection

Enabling a connection makes it available for user authentication with the provider and for use in the Integration node on the workflow canvas. Use the toggle to enable (default) or disable the connection as needed.

## Integrate with Weights & Biases

Connecting to Weights & Biases (WandB) links AI for Process to your WandB account so that fine-tuning data is sent to the WandB console for additional analytics.

<Steps>
  <Step title="Access Integrations">
    In AI for Process, click **Settings** > **Integrations**.
  </Step>
  <Step title="Select Weights & Biases">
    Click **Weights & Biases** from the integrations list. The section expands.
  </Step>
  <Step title="Add a connection">
    Click **Add connection**. In the dialog, enter a **Connection name** and your **API key**.
  </Step>
  <Step title="Test and confirm">
    Click **Test** to verify the connection. If successful, click **Confirm** to complete the setup.
  </Step>
</Steps>

<Note>Once the WandB connection is set up, the connection name appears in the **Weights and Biases** section of the Create a Fine-Tuned Model wizard.</Note>

## Integrate with Hugging Face

Connecting to Hugging Face lets you use publicly available text generation models directly in AI for Process. For private or exclusive models, provide your Hugging Face access token.

<Steps>
  <Step title="Access Integrations">

  </Step>
  <Step title="Select Hugging Face">
    Click **Hugging Face** from the integrations list. The section expands.
  </Step>
  <Step title="Add a connection">
    Click **Add connection**. In the dialog, enter a **Connection name** and your **Access token**.
  </Step>
  <Step title="Test and confirm">
    Click **Test** to verify the connection. If successful, click **Confirm** to complete the setup.
  </Step>
</Steps>

<Note>Once the connection is set up, the connection name appears in the dropdown when selecting and deploying an open-source model.</Note>

## Integrate with AWS S3

Connecting to AWS S3 lets you import files from your S3 bucket and use them in AI for Process workflows.

<Steps>
  <Step title="Access Integrations">
    In AI for Process, click **Settings** > **Integrations**.
  </Step>
  <Step title="Select AWS S3 bucket">
    Click **AWS S3 bucket** from the integrations list. The section expands.
  </Step>
  <Step title="Add a connection">
    Click **Add connection** and enter:
    - **Connection name**: A unique name for the connection.
    - **Access key**: The unique identifier for your AWS account.
    - **Secret key**: The confidential key paired with the Access Key ID.
    - **Bucket name**: The bucket name configured in the S3 console.
  </Step>
  <Step title="Test and confirm">
    Click **Test** to verify the connection. If successful, click **Confirm** to complete the setup.
  </Step>
</Steps>

### Use S3 files in the Workflow Builder

Once the S3 connection is set up:

1. Create an **Input variable** with type **Remote File**.
2. Add an API node and point to the remote file using the **URL** field.

The file content is available in the context object and can be accessed in any subsequent nodes.
