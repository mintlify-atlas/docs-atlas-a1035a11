The Agent Platform provides two sets of APIs:

- **Agentic App APIs**: The Agent Protocol API enables programmatic interaction with your agents — managing conversational sessions, executing agent logic, retrieving responses, and tracking asynchronous tasks.
- **Tools APIs**: Secured APIs to manage files, tools, models, and guardrails. Developers can perform key activities like deployment, import/export, monitoring, analytics, billing, and reporting.

## Authentication

### Agentic App APIs

All Agentic App API requests are made to `https://<your-agent-platform-env>.<domain>.com/api/v2/`. All requests must include an API key in the `Authorization` header.

<Note>API keys are app-specific. They're not shared across accounts or workspaces. Use the API key associated with the correct app when sending requests.</Note>

### Tools APIs

Public APIs require an API key for authentication and authorization. The account owner or admin can generate this key in the **Settings** console.

An API key:
- **Authenticates requests** — verifies the identity of the client
- **Controls access** — restricts API usage based on permissions or rate limits
- **Monitors usage** — tracks API calls for analytics, billing, or debugging
- **Prevents abuse** — protects against unauthorized or excessive use

<Note>Each API key includes created on and last used dates to track its usage history.</Note>

### How to create an API key

<Steps>
  <Step title="Log in and access settings">
    Log in to the Platform and access the Settings Console.
  </Step>
  <Step title="Navigate to API Scopes">
    On the left menu, click **Security & Control** > **API Scopes**.
  </Step>
  <Step title="Create an API app">
    Create an API app, include all the required scopes, and generate the API key.
  </Step>
</Steps>

<Warning>You can copy the API key only once. Save the key in a secure location for future use.</Warning>

## Key concepts

| Term | Definition |
|---|---|
| **App (AppId)** | Represents a specific app created for the Platform |
| **Environment (envName)** | Refers to a deployment environment for the app. The environment name must match the exact name as configured in the application. |
| **Session (sessionId)** | A unique ID representing a single conversation between a user and the agent. Sessions allow multi-turn conversations by preserving context. |
| **Run (runId)** | Represents a single execution step within a session, typically triggered by user input |

### Session identity fields

| Field | Priority | Purpose |
|---|---|---|
| `sessionId` | Highest | Used only to retrieve an existing session. Cannot be used to create new sessions. |
| `sessionReference` | Medium | Attempts to find an existing session; creates a new one if not found. |
| `userReference` | Lowest | Always creates a new session associated with the specified user. |

## Session resolution

Session resolution follows a **priority-based approach**. When more than one identifier is provided, the Platform checks them in order of priority, starting with the highest.

- If `sessionId` is provided, the Platform retrieves the existing session (cannot create new).
- If `sessionReference` is provided and a matching session exists, it is retrieved; otherwise a new session is created.
- If only `userReference` is provided, a new session is always created.

## OAuth authorization

If an agent includes tools that require OAuth authorization, the Platform ensures authorizations are completed before API calls are made.

At the **first invocation**, if any required authorization is pending, the API returns an `IDP_Redirect` event:

```json
{
    "messageId": "msg-260f4d3c-5b8c-4056-af97-11317fc28c8d",
    "events": [
        {
            "type": "IDP_Redirect",
            "content": {
                "auth_profiles": [
                    {
                        "url": "https://{host}/r/396c63515671634648357955",
                        "idpName": "Google",
                        "isAuthorized": false,
                        "sso_type": "oauth2"
                    }
                ]
            }
        }
    ],
    "output": [
        {
            "type": "text",
            "content": "Hello! How can I assist you today?"
        }
    ],
    "sessionInfo": {
        "status": "idle",
        "sessionId": "s-54f40bda-1505-4f9e-b38e-88d39ea36d58"
    }
}
```

Go to the URL in the events object to complete authorization, then reinvoke the API.

<Info>Authorization is only required once per user (across sessions) or until the token expires.</Info>

## API reference

### Agentic App APIs

| Use case | API |
|---|---|
| Create, retrieve, or end a session with an agent | Sessions API |
| Send user input to an agent and receive the agent's response | Runs API |
| Upload or delete attachments to a conversation | File Attachment APIs |
| Create a new Agentic App | Create App API |

### File management

| Use case | API |
|---|---|
| Upload a small or large public file in supported formats | File Upload API |

### Process status

| Use case | API |
|---|---|
| Check the status of an ongoing or completed job related to tools or models | Get Dock Status API |

### Manage tools

| Use case | API |
|---|---|
| Import a new tool | Import a New Tool API |
| Import new configurations or updates into a tool | Import to an Existing Tool API |
| Export a tool's configuration for backup or reuse | Export a Tool API |
| Deploy a specific tool into an environment | Deploy a Tool API |
| Undeploy a specific tool from an environment | Undeploy a Tool API |

### Manage models

| Use case | API |
|---|---|
| Import a model in chunks | Import a Model API |
| Export a trained AI model | Export a Model API |
| Deploy a model into the environment | Deploy a Model API |
| Undeploy a model from the environment | Undeploy a Model API |
| Manage external model connections | External Model Connection APIs |
