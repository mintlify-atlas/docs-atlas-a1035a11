AgenticAI Core is a Python SDK for building and deploying multi-agent AI applications. You define agents, tools, models, and orchestration in Python (**design-time**) and execute them through a runtime that exposes an MCP server.

Typical uses include customer-facing assistants, workflow automation, and domain-specific AI systems that require multiple specialized agents.

## Architecture and core concepts

### Design-time and runtime

Design-time focuses on describing *what exists* in your application, while runtime focuses on *how requests are executed*.

```
Design-Time  ->  App + Agents + Tools + Models
Runtime      ->  MCP Server -> Orchestrator -> Agents -> Tools
```

### Main building blocks

| Building block | Description |
|---|---|
| **App** | The top-level container that groups agents and defines orchestration |
| **Agents** | Perform domain-specific work and may reason (REACT) or proxy to external systems |
| **Tools** | Callable actions, typically Python functions registered with `@Tool.register` |
| **LLM models** | Define how agents reason |
| **Prompts** | Constrain agent behavior |
| **Memory stores** | Persist context across interactions |
| **Orchestration** | Controls routing between agents |

A request flows as:

```
Client -> MCP Server -> Orchestrator -> Agent -> (LLM <-> Tools) -> Response
```

## Installation and setup

### Prerequisites

- Python 3.10+
- pip
- Git

The SDK is provided through a private workspace repository. [Contact support](https://support.kore.ai) to get access.

### Workspace setup

```bash
git clone <workspace-repository-url>
cd workspace
chmod +x .setup.sh
./.setup.sh
```

The script creates a `.venv` virtual environment and installs `agenticai-core` and its dependencies.

### Platform configuration

Create environment files for each deployment environment:

```bash
# .env/dev
KORE_HOST=https://agent-platform.kore.ai
APP_API_KEY=your_api_key
TRACING_ENABLED=True
```

### Workspace structure

```
workspace/
├── .venv/                    # Virtual environment
├── .env/                     # Environment configurations
│   ├── dev
│   ├── staging
│   └── prod
├── lib/                      # Pre-installed libraries
│   ├── agenticai_core-0.1.0-py3-none-any.whl
│   └── kore_api-1.0.0-py3-none-any.whl
├── src/
│   ├── tools/                # Your custom tools
│   ├── orchestrator/         # Your custom orchestrators
│   └── app.py                # Your application definition
├── bin/                      # Generated archives
├── examples/                 # Example applications
├── requirements.txt
├── run.py                    # CLI entry point
└── .setup.sh                 # Setup script
```

## First application

<Steps>
  <Step title="Define a tool">
    ```python
    from agenticai_core.designtime.models.tool import Tool

    @Tool.register(name="Get_Balance", description="Get account balance")
    async def get_balance(account_id: str):
        return {"balance": 1000}
    ```
  </Step>
  <Step title="Define an agent and app">
    ```python
    from agenticai_core.designtime.models import App, Agent

    app = App(
        name="Sample App",
        agents=[Agent(name="DemoAgent")]
    )
    ```
  </Step>
  <Step title="Run locally">
    ```bash
    source .venv/bin/activate
    python run.py start -H localhost -P 8080
    ```
  </Step>
  <Step title="Package and deploy">
    ```bash
    python run.py package -o sample-app
    python run.py config -u dev
    python run.py deploy -f bin/sample-app/application.kar
    ```
  </Step>
</Steps>

## Development lifecycle

```
Clone -> Setup -> Develop tools -> Test Local -> Package -> Deploy -> Test E2E -> Monitor
```

## Best practices

- Keep agents narrowly focused on a single domain
- Design tools to do one thing well
- Test locally before every deployment
- Use `.venv` consistently to avoid oversized packages
- Save deployment identifiers returned by the Platform for later testing
