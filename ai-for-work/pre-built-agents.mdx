## Publish a pre-built agent

Administrators can publish selected pre-built agents to make them available to users through the Agent Store.

<Steps>
  <Step title="Navigate to pre-built agents">
    Go to **User Profile** > **Admin Console** > **Workspaces** > **API Agents** or **Prompt Agents**. You'll see a list of Pre-Built Agents, each marked with a "Prebuilt" label.
  </Step>
  <Step title="Enable the agent">
    Toggle the switch **ON** next to the specific agent, or click the **ellipsis (⋮)** icon. A pop-up window appears.
  </Step>
  <Step title="Configure publishing settings">
    Provide the following details:

    - **Published Version**: Select the version of the agent you want to publish.
    - **Publish to**: Choose who has access to the agent:
      - **Admins**: Restricts the agent to Admin users only.
      - **Selected User Groups/Users**: Allows you to specify individual users or groups.
      - **Everyone in the Account**: Makes the agent available to all users.
    - **Actions**: Choose what users can do with this agent:
      - **Lookup**: Users can search for existing items.
      - **Creation**: Users can create new items.
  </Step>
  <Step title="Configure authentication">
    Click **Publish**. The system displays the **Auth Settings** screen before completing the publishing process. Configure the authentication method required for the agent.
  </Step>
</Steps>

## Configure authentication settings

When you click **Publish** for a pre-built agent, the platform presents the Auth Settings screen. You must configure authentication before the agent is available to users.

### Authorization methods

The Auth Settings screen displays the type of authorization the agent supports:

- **OAuth**: Most pre-built agents use this method (Jira, HubSpot, Microsoft 365 Calendar, Microsoft 365 Email, Microsoft OneDrive, Microsoft Teams, Google Calendar, Google Mail, Google Drive, Slack).
- **Basic Auth**: The Zendesk agent uses this method.

### OAuth application configuration

<Tabs>
  <Tab title="Use product registered app">
    Select **Use Product Registered OAuth Application** to route the OAuth credentials of the user's account to the product-registered application. The platform stores and manages the credentials automatically.
  </Tab>
  <Tab title="Use your own OAuth app">
    Select **Use Your Own OAuth Application** to use your organization's own OAuth application credentials. Provide:

    - **Client ID**: Enter the Client ID from your OAuth application.
    - **Client Secret**: Enter the Client Secret from your OAuth application.
    - **Redirect URL** (read-only): Register this URL in your OAuth application.
    - **Scopes** (read-only): Required OAuth scopes for the agent.

    The **Save** button enables after you enter valid values for Client ID and Client Secret.
  </Tab>
</Tabs>

<Note>For Google agents (Google Calendar, Google Mail, and Google Drive), you must provide your own OAuth application credentials.</Note>

## Using a published agent

After the administrator publishes an agent and configures authentication, users can access it through the Agent Store.

<Steps>
  <Step title="Open the Agent Store">
    Users navigate to the Agent Store from the AI for Work homepage.
  </Step>
  <Step title="Connect to the agent">
    Click **Connect** for the desired agent and follow the on-screen instructions to **Add Connection**.
  </Step>
  <Step title="Start using the agent">
    After successfully adding the connection, the agent responds to user questions and queries through the **Ask or Search Anything** box on the homepage.
  </Step>
</Steps>
